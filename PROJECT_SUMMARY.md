# BeautySID-Rec: 基于语义ID的生成式推荐项目总结

## 1. 数据集与统计分析

本次复现使用 Amazon Review 的 Beauty 子集，经过 5-core filtering 后得到更稳定的训练数据。数据集核心统计如下：

| 统计项 | 数值 |
|---|---:|
| 用户数 | 22,363 |
| 商品数 | 12,101 |
| 总交互数 | 198,502 |
| 平均每用户交互数 | 8.88 |
| 用户序列中位数长度 | 4 |
| 用户序列 90 分位长度 | 12 |
| 用户序列 95 分位长度 | 19 |
| 最短/最长序列 | 3 / 202 |
| 数据密度 | 0.0734% |
| 数据稀疏度 | 99.93% |

训练/验证/测试均按时间顺序切分：最后 1 个 item 为 test，倒数第 2 个为 valid，其余为 train。训练阶段启用 sliding-window 增强后，实际可用训练样本数约为 153,776。

## 2. 模型架构与关键参数

### 2.1 文本编码器

当前使用的是 `BAAI/bge-base-en-v1.5`，用于把商品文本转成 768 维语义向量。

| 参数 | 作用 |
|---|---|
| `model_name` | 选择预训练文本编码器 |
| `batch_size` | 影响编码速度与显存占用 |
| `max_length` | 文本截断长度，影响语义保留 |
| `device` | 编码运行设备 |

embedding 生成统计：

| 统计项 | 数值 |
|---|---:|
| embedding 维度 | 768 |
| 单商品平均编码耗时 | 0.0071 s |
| 总耗时 | 85.57 s |

### 2.2 语义 ID / 码本设计

采用 RQ-VAE 风格 tokenizer，将每个商品编码为 4 层语义 ID，并额外追加 1 个 de-dup token。

| 参数 | 当前设置 | 含义 |
|---|---|---|
| `num_layers` | 4 | 语义量化层数 |
| `codebook_sizes` | [16, 32, 64, 128] | 每层码本大小 |
| `append_dedup_token` | true | 是否追加冲突消解 token |
| `dedup_default_token` | 0 | 无冲突时默认值 |
| `normalize_residuals` | true | 归一化残差，防止退化 |

### 2.3 生成模型

采用轻量 encoder-decoder Transformer 作为 next-item 生成模型。

| 参数 | 当前设置 | 含义 |
|---|---|---|
| `d_model` | 320 | token embedding / hidden 维度 |
| `n_heads` | 8 | 多头注意力头数 |
| `num_encoder_layers` | 4 | 编码器层数 |
| `num_decoder_layers` | 4 | 解码器层数 |
| `dim_feedforward` | 1024 | FFN 宽度 |
| `dropout` | 0.08 | 防过拟合比例 |
| `max_seq_len` | 50 | 最大历史长度 |

## 3. 码本冲突分析

码本冲突是指：多个商品在前 4 层语义 ID 上完全相同，导致它们共享同一个“基础 SID”。这会让模型在检索/生成时无法仅凭前 4 位区分这些商品。

### 3.1 当前冲突定义

- **基础冲突**：只看前 4 层 SID 时，多个商品落入同一编码。
- **最终冲突**：加入第 5 位 de-dup token 后，是否仍然有重复 SID。

### 3.2 冲突变化表

| 配置 | 商品总数 | 基础唯一 SID | 基础冲突商品数 | 基础冲突率 | 最终唯一 SID | 最终冲突商品数 | 最终冲突率 |
|---|---:|---:|---:|---:|---:|---:|---:|
| 4 层 RQ-VAE（无 de-dup） | 12,101 | 9,685 | 4,058 | 33.53% | - | - | - |
| 4 层 RQ-VAE + de-dup | 12,101 | 9,685 | 4,058 | 33.53% | 12,101 | 0 | 0.00% |

### 3.3 码本利用率

| 层数 | 码本大小 | 实际使用 | 利用率 |
|---|---:|---:|---:|
| 第1层 | 16 | 15 | 93.75% |
| 第2层 | 32 | 26 | 81.25% |
| 第3层 | 64 | 29 | 45.31% |
| 第4层 | 128 | 25 | 19.53% |
| de-dup 层 | 17 | 17 | 100.00% |

### 3.4 冲突处理策略

1. **多层残差量化**：把语义信息拆成 4 层 token，兼顾表达能力和可学习性。  
2. **de-dup token**：当多个商品在前 4 层 SID 相同的时候，用第 5 位做局部区分。  
3. **默认 token 归零**：无冲突商品不额外增加复杂度。  
4. **保留原始冲突文件**：便于后续做消融和回溯分析。

## 5. 与论文设置的差异

| 项目 | 论文思路 | 当前复现 |
|---|---|---|
| 文本编码器 | 更偏实验设置化 | `bge-base-en-v1.5`，更强且稳定 |
| SID 设计 | 多层语义 ID | 4 层语义 + 1 层 de-dup |
| 解码 | 常见 beam search / constrained decoding | 先用普通 beam search 跑通 baseline |
| 训练目标 | SFT + RL（论文完整版） | 先做可跑通 baseline，未纳入 RL |
| 结果侧重 | 追求论文对齐 | 更关注稳定复现、可扩展和后续消融 |

## 6. 项目亮点

- 完成了 Beauty 数据的自动处理、文本编码、RQ-VAE tokenizer、SID 导出、训练和测试闭环。
- 通过 `de-dup` 机制把原本较高的 SID 冲突降到 0。
- 结果相较早期版本明显提升，`Recall@10` 达到 0.0371，`NDCG@10` 达到 0.0179。
- 代码结构模块化，便于后续替换 tokenizer、编码器、Transformer 和解码策略。

## 7. 可用于简历的项目总结

BeautySID-Rec 是一个面向 Amazon Beauty 数据集的生成式推荐复现项目，我把传统“商品 ID + embedding”的方案改造成了“文本语义编码 + RQ-VAE 语义 ID + encoder-decoder Transformer”的完整流水线。数据层面，项目处理了 22,363 个用户、12,101 个商品和 198,502 条交互，数据极度稀疏，密度仅 0.0734%。模型层面，我用 `BAAI/bge-base-en-v1.5` 提取 768 维语义向量，再通过 4 层残差量化码本 `[16,32,64,128]` 生成语义 ID，并增加第 5 个 de-dup token 解决 SID 冲突，最终将商品级冲突率从 33.53% 降到 0。推荐模型使用 4 层 encoder-decoder Transformer，在测试集上取得 `Recall@10=0.0371`、`NDCG@10=0.0179`。相比前一版实现，效果有明显提升，说明更强文本编码器、合理码本设计和冲突消解机制对生成式推荐性能具有直接增益。整个工程保留了完整的 profiling、metrics 和可配置结构，后续可以直接扩展做 tokenization、beam search、encoder-decoder 结构以及 RL 消融实验。
